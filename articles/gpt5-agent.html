Windows Copilot 被逆向！无需 API Key 免费白嫖 GPT-5，对接本地 AI Agent

就在最近，GitHub 上出现了一个令人振奋的开源项目——开发者成功将 Windows Copilot 封装成了 OpenAI 兼容 API。这意味着用户可以直接免费使用 GPT-4o、GPT-5.5 等模型，甚至可以对接本地 AI Agent 来执行自动化任务。

🎯 这个项目能做什么？

这个项目名为 Windows-Auto-Copilot，它将 Windows 系统中隐藏的 Copilot 能力提取出来，以标准 OpenAI API 的形式暴露给第三方应用。项目支持以下功能：

  • ✅ 无需 OpenAI API Key
  • ✅ 无需购买 ChatGPT Plus 订阅
  • ✅ 支持 OpenAI 标准接口（/v1/chat/completions）
  • ✅ 可直接接入 Cherry Studio、Open WebUI 等客户端
  • ✅ 支持本地 Agent 对接（AutoGPT、LangChain 等）
  • ✅ 代码生成能力相当不错

🚀 快速部署教程

环境要求

  • Windows 11 23H2 及以上版本
  • 已启用 Windows Copilot 功能
  • Python 3.10+
  • 至少 8GB 内存

步骤一：下载项目

git clone https://github.com/your-repo/Windows-Auto-Copilot.git
cd Windows-Auto-Copilot
pip install -r requirements.txt

步骤二：启动服务

python server.py --port 8080

服务启动后，默认会在 http://localhost:8080 运行，API 地址为 http://localhost:8080/v1/chat/completions。

步骤三：对接 Agent

以 Cherry Studio 为例，在设置中填入以下地址即可：

API 地址: http://localhost:8080/v1
模型名称: gpt-5.5-preview
API Key:（留空或填任意字符）

⚠️ 注意事项

  • 该方法仅供学习研究使用，请遵守微软服务条款
  • 部分地区可能无法正常使用 Windows Copilot
  • 建议配合 VPN 使用以获得更稳定的连接
  • 项目代码已开源，学习成本低，有能力的开发者可自行 fork 二次开发

💡 使用场景推荐

结合本地 Agent 使用，你可以实现：

  • 自动化办公：让 AI 自动处理邮件、生成报告
  • 代码审查：GPT-5 的代码能力配合本地 Agent 自动审查 PR
  • 数据分析：对接本地知识库进行复杂数据分析
  • 智能助手：打造个人专属的 AI 助手
